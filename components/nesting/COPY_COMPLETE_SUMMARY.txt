🚀 QUICK START
================================================================================

1. IMPORT:
   import { DrawingWorkspace } from '@/components/nesting';

2. USE IN COMPONENT:
   const [cadEntities, setCadEntities] = useState([]);
   const [activeTool, setActiveTool] = useState(null);

   return (
     <DrawingWorkspace
       activeDrawTool={activeTool}
       onToolChange={setActiveTool}
       onCadEntitiesChange={setCadEntities}
       width={1200}
       height={600}
     />
   );

3. PROCESS ENTITIES:
   cadEntities.map(entity => {
     console.log(entity.type);      // 'line', 'rect', 'circle', etc.
     console.log(entity.points);    // Array of {x, y}
     console.log(entity.properties);// Additional data (radius for circle)
   });

📊 COMPARISON: BB1 vs AX Implementation
================================================================================

BB1 APPROACH (Original):
  • Monolithic Workspace component (1500+ lines)
  • Mixed concerns (UI + Drawing + Nesting)
  • Hard to reuse
  • Tightly coupled to NestingAX App

AX NESTING APPROACH (New):
  ✅ Modular components (separate files)
  ✅ Reusable DrawingWorkspace
  ✅ Standalone DrawingTools
  ✅ Clean separation of concerns
  ✅ Multiple integration examples
  ✅ Better documentation
  ✅ Flexible and extensible
  ✅ TypeScript-first design
